## Done / answered

1. **elegant scope.** elegant is used only as the S-band linac reference
   (RFCA + TWLA, 0.06% vs RF-Track). It is not needed for the diagnostic
   chicane (COSY/xsuite/FELsim cover it; CSR is negligible at 40 MeV) and is
   not wired anywhere else. Proposed next use: an elegant cross-check of the RF
   gun (currently RF-Track-only). *Confirm: is that the scope you meant, and
   do you want the gun cross-check?*

2. **Linac codes x physics-feature table.** New table built (codes x TW/SW/
   field-maps/multi-cell/SC/beam-loading/wakefields/longitudinal/autophasing/
   det(R), each cell implemented-vs-potential with file-level evidence). RF-Track
   is the only cross-validated linac model; xsuite is a lumped cavity; COSY
   RFCAVX_TM (SW) exists upstream but is not yet wired into the COSY adapter.
   See `linac_codes_physics_matrix.md`.

4. **PALS beam-file standard? No.** PALS is a lattice standard only; the
   particle-distribution format is ours to choose. Recommendation:
   openPMD-beamphysics (HDF5) as the hub interchange (reads ASTRA / SDDS-elegant
   / Genesis4), plain ASCII as fallback. Genesis4 is HDF5-native both ends, which
   favours this. See `pals_beam_file_standard_memo.md`.

5. **Fringe-field (FF) mode comparison.** COSY reaches the undulator match in
   every fringe order FR0/FR1/FR2/FR3 (beta_y = 0.242 m, MSE < 1e-8 warm-started),
   each with a different current set (RMS shift 0.19/0.65/1.08 A vs FR0). FELsim
   and COSY agree on matched Twiss (R2); the R2-era RF-Track beta_y deficit
   (0.055 vs 0.242) is closed by the triangle-rule edge correction added since
   (RFT-opt MSE 7.0e-3). See `ff_mode_comparison.md`. Remaining: re-optimise
   RF-Track + xsuite to the target for a single 5-code FF figure.

6. **Cross-calibration.** Backbone verified across all four codes: one anchor
   G = 2.694 T/A/m, identical current->k1 formula, consistent delta conventions
   (COSY dK/K0 vs xsuite dp/p0), agreement to the test tolerances (xsuite
   round-trip 5.5e-13, FODO 7.5e-5; 3-code Twiss at eps_n=8). See
   `cross_calibration_status.md`.

8. **glyfada.** Already tried and characterised (W6/W7/O4/O5): glyfada's
   evolutionary algorithms underperform NM+pycma by 3-10 orders on this
   narrow-basin objective, so production is two-phase NM -> pycma CMA-ES. The one
   worthwhile fresh run is glyfada/global-search on the Config-C stress case for
   the paper. See `glyfada_optimization_findings.md`.
