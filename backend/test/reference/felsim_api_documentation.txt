result = test.calc("Nelder-Mead", segmentVar, startPoint, objectives)

Parameters:
                        Method: String name of scipy.optimize method for minimization
Ex. “Nelder-Mead”, “COBYLA”, see https://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.minimize.html


segmentVar: dictionary of the relationships between x variables to optimize. Each integer key corresponds to the end of a specified interval within the beamline list to optimize. Each list value pair has information for x variable, 


Ex. [str variable name, str segment attribute to optimize] 
If you want to optimize indice 2’s length, quadruple at indice 1’s current, and dipole at indice 3’s angle, format would follow:
segmentVar = {2: ["L","length", lambda num:num],
1: ["I", "current", lambda num:num],
3: ["A", "angle", lambda num:num],...}
                        Example 1.5


                        Segment values can be mathematically related, such as:
segmentVar = {5: ["I", "current", lambda num:num],
1: ["I", "current", lambda num:num/np.sqrt(2)],
3: ["I", "current", lambda num:num**2],...}
Example 1.6


For any manually beam segment types added in the beamline class, as long as segment’s attribute exist, program can automatically optimize it


startPoint: dictionary of parameters for x variables to optimize. Each key string representation of a x variable corresponds to its value dictionary containing start and bound parameters for that variable. Start is the float value function begins looking at, bounds are the numeric bounds software is restricted to for that variable.


Ex. 
{str x variable: {“bounds”: (float lower bound, float upper bound), “start”: float algorithm start search point}}
Say we have 2 x variables I and L. If we want software to find L values between 0-1 and start search at 0.1, and find I values between 0-10 and start search at 5, format startPoint as follows:


startPoint = {"I": {"bounds": (0,10), "start": 5},
           "L": {"bounds": (0, 1), "start": 0.1}}
Example 1.7


objectives: dictionary about statistical y objectives to optimize for at THE END of a beam segment. Each integer key corresponds to a list of dictionaries containing parameters for each statistical y objective. 


Ex. 
{str indice: [“measure”: [str particle variable, str method name/method object], “goal”: float y objective value, “weight”: float y objective statistical weight}


If you want program to optimize for x alpha = 0 and y gamma = 1 at indice 5, x standard deviation = 1 and y standard deviation = 1 at indice 9, use:


objectives = {5:[{"measure": ["x", "alpha"], "goal": 0, "weight":1},
                {"measure": ["y", "gamma"], "goal": 1, "weight":1}],
             9:[{"measure": ["y", "std"],"goal":1,"weight":1},
                {"measure": ["x", "std"],"goal":1,"weight":1}]}
Example 1.8


For “measure” key, the method described as the second list element must be found in beamOptimizer.py’s self.OBJECTIVEMETHODS dictionary located in the constructor


You can also add your own defined function object as long as it follows y objective function format:
def alpha(particles, variable):
       ebeam = beam()
       dist_avg, dist_cov, twiss = ebeam.cal_twiss(particles, ddof=1)
       return twiss.loc[variable].loc[r"$\alpha$"]


objectives = {7:[{"measure": ["x", alpha], "goal": 5, "weight":1},
                {"measure": ["y", alpha], "goal": 5, "weight":1}]}

        getTwissSigmai():
